## MiroFish AI Security Analysis (21:41 CDT)

Rusty asked me to analyze **MiroFish AI** (mirofish.ai) for vulnerabilities.

### What It Is
- Multi-agent swarm intelligence platform for social media opinion simulation
- "Predict Anything" - simulates public opinion, events, scenarios
- Built on OASIS simulation engine (CAMEL-AI)
- Uses Zep Cloud for GraphRAG memory
- Backed by Shanda Group (盛大集团) - Chinese company
- Open source: github.com/666ghj/MiroFish

### Infrastructure Found
| Component | Details |
|-----------|---------|
| Frontend | Cloudflare Pages (`mirofish-demo.pages.dev`) |
| Backend | Supabase (`mfnhvuvhgqkfkzprrqxf.supabase.co`) |
| GitHub | `github.com/666ghj/MiroFish` (public) |
| Tech Stack | Vue.js/Vite frontend, Python Flask backend |

### Security Findings

**HIGH SEVERITY:**
1. **Error traceback exposed to clients** - `traceback.format_exc()` sent in API responses, leaks internal code paths
2. **No authentication on API endpoints** - Anyone can delete projects, run simulations, access documents

**MEDIUM SEVERITY:**
3. **Hardcoded default SECRET_KEY** - `'mirofish-secret-key'` allows session/CSRF forgery
4. **Wide-open CORS** - `origins: "*"` enables CSRF attacks
5. **Debug mode defaults to True** - Werkzeug debugger potentially exposed
6. **File upload only checks extension** - No content validation

**LOW SEVERITY:**
7. No rate limiting (denial of wallet attack possible)
8. Predictable resource IDs
9. Supabase anon key in JS bundle (expected, but should verify RLS)
